Fortis Title & Escrow · AI Setup Runbook

Getting Claude working for Fortis, step by step

Work top to bottom. Every gray box is a prompt — hit Copy, paste it into Claude, and follow what comes back. Phases 1–3 happen today; Phases 4–6 are this week.

Two rules before anything else. (1) No client Social Security numbers, bank account numbers, or wire instructions go into any AI chat — ever. Names, addresses, file numbers, and stages are fine. (2) The AI drafts; a person sends. It never states binding closing figures and never touches money.

  • Download the Claude desktop app from claude.ai/download, install it, and sign in with the same account. This is where all the day-to-day work happens — pin it to the dock or taskbar so it's one click away.

  • Grab the Claude app on Mike's phone too, same login — the Project and its rules follow him to closings.

Phase 4 · The command center~1 hr, this week

Build the Airtable hub

One dashboard that answers: what's happening, what needs me today, and how's marketing. Create a free Airtable account at airtable.com (same work email), then paste this into Claude — it walks you through the whole build and generates the starter files.

Prompt · Build the base
I'm building an Airtable base called "Fortis Command Center" for our title agency. Walk me through it step by step, one table at a time, and wait for me to say "done" before moving to the next. At the end, generate an importable starter CSV for each table with 3 example rows I can replace with real files.

The schema:

TABLE 1 — Open Files (the core pipeline, one row per closing):
File # (primary, our Qualia order ID), Property Address, Stage (single select: Order Received / Title Search / Commitment Issued / Clear to Close / Scheduled / Closed / Post-Closing / On Hold), Transaction Type (Purchase / Refi / Commercial / Construction), Closing Date (date), Title Commitment Due (date), Earnest Money Received (checkbox), Buyer and Seller (links to Parties), Referral Source (link to Referral Sources), Assigned To (single select of our staff), Wire Verified (checkbox), Days in Stage (formula from a Last Stage Change date field), Needs Attention (formula: TRUE if closing within 3 days and not yet Clear to Close or Scheduled, OR more than 7 days in stage, OR no earnest money past Order Received), Notes.

TABLE 2 — Parties: Name, Role (Buyer/Seller), Email, Phone, Military (checkbox), linked Files.

TABLE 3 — Referral Sources: Name, Type (Agent / Loan Officer / Builder / Attorney / Other), Company, Email, Phone, Orders YTD (rollup count from Open Files), Last Order (rollup max date), Last Touch (date), Notes.

TABLE 4 — Leads: Name, Source (Order Form / Net-Sheet / Quote / Call / Other), Received (date), Status (New / Working / Won / Lost), Worked By, Converted to File (link), Notes.

TABLE 5 — Reviews: Reviewer, Platform (Google / Facebook / Other), Rating, Date, Requested (checkbox), Posted (checkbox).

Views to build: a Kanban of Open Files grouped by Stage; "What Needs Me Today" (Needs Attention = TRUE, sorted by Closing Date); "Closing This Week" (calendar); a Referral Leaderboard sorted by Orders YTD.

After the tables and views work, walk me through building one Airtable Interface page: a dashboard showing the pipeline Kanban, the What Needs Me Today list, and the closing-this-week calendar, so the team sees a clean screen instead of raw tables.

Important: before we start, ask me for the exact order status names we use in Qualia so the Stage options match what my processors already see.

What the team does to keep it current

  • Processors: new order → add a row (2 minutes). Stage changes → update one dropdown (10 seconds). Tick earnest-money and wire-verified boxes as they happen.
  • Front desk / inbox: new web lead or quote request → add a Leads row (1 minute).
  • One named owner: 5-minute end-of-day sweep for stale stages, plus a 15-minute Friday cross-check against a Qualia export.

Total across the whole team: about 10–15 minutes a day. Mike maintains nothing — he reads it.

Send this regardless of what the console shows — the webhook and pricing answers matter either way, and the response is the slow part.

  1. Generate the API key in Qualia (admin console).

Timeline: if the console already shows API access, this is a few focused hours — realistically a second visit this week. If it needs a plan upgrade, expect one to three weeks, mostly waiting on the rep, and the Friday export bridge covers the gap. This step is Patrick's build, not Mike's — Mike's only jobs are the rep email and the yes/no on any API-tier cost.

Phase 7 · Later — Coworka few weeks in, once the habits stick

The upgrade: Claude that works with your files

Everything above happens in a chat window: you bring things to Claude by pasting or attaching them. Cowork is the next level. It's a mode in the Claude desktop app where Claude can open a folder on the computer, read what's inside, and produce real documents — Word files, spreadsheets — instead of just chat replies.

In plain terms: today you copy things to Claude. With Cowork, Claude comes to your files.

What it's for at Fortis

  • Check a closing folder. Point it at one file's folder and ask: "Read everything in here. What's missing, and what deadlines do these documents create?"
  • Skip the export step. With the Airtable connector switched on, ask "what needs me today?" and Claude reads the command center directly — no CSV.
  • Make real documents. The weekly marketing readout as a Word doc, a batch of template letters, a summary spreadsheet — finished files, not text to copy out of a chat.

Why wait a few weeks

Two reasons. First, one new tool per visit is the pace that sticks — the chat habits come first. Second, Cowork can see files, and closing folders contain Social Security numbers and bank details. Before it ever touches real client files, the boundaries below get set on purpose, not in a hurry.

Setup, when the time comes

  • The desktop app is already installed from Phase 1 — nothing is rebuilt; the Project, its instructions, and the prompt library carry over.
  • Open Cowork mode in the app (this is the step that waited).
  • Make one folder on the computer called Claude Workspace. That is the only folder Claude gets pointed at. To have Claude look at a closing file, copy that file's documents into the workspace first — client folders themselves stay off limits.
  • Switch on the Airtable connector in the app's settings and connect the Fortis Command Center base.
  • Run the first task together on a closed file (paste the prompt below), so the first live test has nothing at stake.
Prompt · First Cowork task (use a closed file)
The folder I've given you contains the documents from one of our closed title files. Read everything in it and give me: (1) a list of each document and what it is, in one line each; (2) anything a title agency's checklist would say is missing from a complete file; (3) every date or deadline these documents created while the file was open. Do not repeat any Social Security numbers or bank account numbers in your answer. Save the result as a Word document called "File Review" in the same folder.

The rules, extended for Cowork

  • Claude only ever works inside the Claude Workspace folder. Never point it at the whole computer or the client drive.
  • Same rule as chat: it can read documents that contain sensitive numbers, but its write-ups never repeat SSNs, account numbers, or wire instructions.
  • It reviews and drafts. A person still checks the work and sends anything that leaves the building.
Standing rules

The guardrails

  • AI drafts and surfaces. A human decides and sends. Every time.
  • Never in a chat: SSNs, bank or account numbers, wire instructions.
  • Never anything of value to a referral source — for referrals or reviews. Reviews are asked for honestly, nothing offered.
  • Referral and marketing data stays internal.
  • Before the automated version goes live, run the plan past counsel and the underwriter.
